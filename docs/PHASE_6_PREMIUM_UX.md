# Phase-6 Premium UX Contract: Clarity & Calm

## 1. Phase-6 Objective
The goal of Phase-6 is to transition Monetarz from an MVP into a premium-grade product.

- **Premium ≠ More Features**: We will not add complexity for the sake of "value."
- **Premium = Clarity, Comparison, Calm**: The value is in the high-quality synthesis of data, making comparisons effortless and maintaining a calm, focused environment.

## 2. Hard UX Constraints (Non-Negotiable)
To maintain the "Calm" design philosophy, the following are strictly prohibited:

- **No Popups**: User flow must never be interrupted by unexpected overlays.
- **No Auto-Alerts**: The system must never "shout" at the user (push notifications/banners) for budget events.
- **No Red/Green Panic Colors**: Standardize on brand HSL palettes (Sage, Moss, Ink). Avoid high-stress semantic colors.
- **No Insight Without Confidence**: If `confidence < 0.5`, the UI must show a "Zero State" (composed silence) rather than speculative data.
- **No New Charts Unless They Reduce Cognition**: A chart is only admitted if it replaces a complex mental calculation with a simple visual pattern.

## 3. Page-Level Intent (Locked)
Each page has a specific, bounded responsibility to prevent information overload.

### Weekly Page
- **Intent**: Show rhythm, trajectory, and intention.
- **Rules**: Must focus on the *current* flow. No historical deep dives or year-over-year comparisons here.

### Calendar Page
- **Intent**: Show spatial density.
- **Rules**: Visualize when and where spending "clusters" occur. No per-day analytics overload or exhaustive transaction lists.

### Analytics Page
- **Intent**: Show bounded comparisons.
- **Rules**: Cross-period comparisons (This Month vs Baseline). Raw tables and long, scrolling lists are forbidden.

## 4. Free vs Pro UX Rules
Revenue is generated by providing deeper context, not by locking core logging tools.

- **Free = Static Snapshots**: Access to "Now" and "Today."
- **Pro = Comparisons, Overlays, Trajectories**: Access to "Vs Usual," trend lines, and predictive paths.
- **Silent Degradation**: If a Pro subscription expires, features must hide gracefully (empty states/simplified views) without "Lock" icons or aggressive upgrade prompts.

## 5. Trust Signal Usage Rules
Internal Phase-5 metadata must be translated into subtle UI affordances.

- **`confidence` → Tone**: High confidence allows for more assertive phrasing; low confidence requires cautious, observational language.
- **`baselineMonth` → Phrasing**: Use phrases like "Since your February baseline..." to anchor the data in time.
- **`sampleSize` → Eligibility Only**: Used only to determine if an insight is eligible to be shown. Never display raw counts like "Based on 3 data points" to the user.
